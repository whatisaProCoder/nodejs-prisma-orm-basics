# Node.js Prisma ORM Basics

This project is a beginner-friendly introduction to using Prisma, a modern database toolkit, with Node.js and TypeScript. It demonstrates basic CRUD (Create, Read, Update, Delete) operations and the fundamental concepts of Prisma ORM.

## About the Project

This project is a simple Node.js application that uses Prisma to interact with a PostgreSQL database. It includes a basic setup for creating and retrieving users and their posts.

### Project Structure

- `prisma/`: Contains the Prisma schema and migration files.
  - `schema.prisma`: Defines the database models and their relationships.
  - `migrations/`: Stores the SQL migration files generated by Prisma.
- `src/`: Contains the source code for the application.
  - `index.ts`: The main entry point of the application.
  - `lib/prisma.ts`: Initializes the Prisma Client.
  - `generated/prisma/`: Contains the auto-generated Prisma Client code.
- `package.json`: Lists the project dependencies and scripts.
- `tsconfig.json`: The TypeScript configuration file.

## Getting Started

These instructions will get you a copy of the project up and running on your local machine for development and learning purposes.

### Prerequisites

- [Node.js](https://nodejs.org/) (v14 or later)
- [npm](https://www.npmjs.com/) (comes with Node.js)
- A PostgreSQL database

### Step-by-Step Setup

#### 1. Initialize the Project

First, we initialize a new Node.js project. This creates a `package.json` file, which will manage our project's dependencies and scripts.

```bash
npm init -y
```

#### 2. Set Up TypeScript

Next, we'll install TypeScript and some related development dependencies. `tsx` allows us to run TypeScript files directly without pre-compiling them, and `@types/node` provides TypeScript type definitions for Node.js.

```bash
npm install typescript tsx @types/node --save-dev
```

After installing TypeScript, we create a `tsconfig.json` file. This file specifies the root files and the compiler options required to compile the project.

```bash
npx tsc --init
```

#### 3. Install Prisma and Database Dependencies

Now, let's install Prisma. We install `prisma` as a development dependency because it's primarily a tool for development, and `@types/pg` for PostgreSQL type definitions.

```bash
npm install prisma @types/pg --save-dev
```

Next, we install the Prisma Client, the PostgreSQL adapter for Prisma, the `pg` driver for PostgreSQL, and `dotenv` for managing environment variables.

```bash
npm install @prisma/client @prisma/adapter-pg pg dotenv
```

#### 4. Initialize Prisma

To start using Prisma, you would typically run `npx prisma init`. This command sets up the Prisma schema file (`prisma/schema.prisma`) and a `.env` file for your database connection string.

You can explore all available Prisma commands by running:

```bash
npx prisma
```

#### 5. Set Up the Database Connection

Make sure your PostgreSQL server is running. Create a `.env` file in the root of the project and add your database connection string:

```
DATABASE_URL="postgresql://user:password@localhost:5432/mydatabase"
```

Replace `user`, `password`, and `mydatabase` with your PostgreSQL credentials and database name.

#### 6. Create the Database Schema

Define your database models in `prisma/schema.prisma`. For this project, we have `User` and `Post` models:

```prisma
// prisma/schema.prisma

model User {
  id    Int     @id @default(autoincrement())
  email String  @unique
  name  String?
  posts Post[]
}

model Post {
  id        Int     @id @default(autoincrement())
  title     String
  content   String?
  published Boolean @default(false)
  author    User    @relation(fields: [authorId], references: [id])
  authorId  Int
}
```

#### 7. Run Database Migration

With the schema defined, we can create and apply a database migration. This command creates the necessary SQL files in the `prisma/migrations` directory and applies them to your database.

```bash
npx prisma migrate dev --name init
```

#### 8. Generate the Prisma Client

After migrating the database, you need to generate the Prisma Client. This command reads your `schema.prisma` file and generates a type-safe client that you can use to interact with your database.

```bash
npx prisma generate
```

The generated client will be placed in `src/generated/prisma`.

## Usage

The main logic of the application is in `src/index.ts`. It demonstrates how to:

- Create a new user with a post.
- Fetch all users and their posts.

To run the application, use the following command:

```bash
npm run dev
```

This will execute the `src/index.ts` file using `tsx`.

## Exploring Your Data with Prisma Studio

Prisma includes a GUI called Prisma Studio that allows you to view and edit the data in your database. To open it, run:

```bash
npx prisma studio
```

This will open a new tab in your browser where you can interact with your data.
